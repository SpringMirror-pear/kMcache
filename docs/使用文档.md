## 4. 安装方式

## 4.1 当前仓库依赖安装

如果你只是想直接运行当前仓库：

```bash
python -m pip install -r requirements.txt
```

## 4.2 本地项目通过源码目录引用

如果你的 FastAPI 项目就在本机上，最简单的方式有两种。

### 方式一：editable 安装

在 `kmcache` 仓库根目录执行：

```bash
python -m pip install -e .
```

然后在你的业务项目中直接：

```python
from kmcache.manager import CacheManager
```

适合：

- 正在并行开发 `kmcache` 和业务项目
- 需要修改库代码后立即在业务项目中生效

### 方式二：本地 wheel 安装

先在 `kmcache` 仓库根目录打包：

```bash
python -m pip wheel . -w dist --no-deps
```

打包后会在 `dist/` 下生成类似文件：

```text
kmcache-0.3.0-py3-none-any.whl
```

然后在你的业务项目环境中安装：

```bash
python -m pip install .\dist\kmcache-0.3.0-py3-none-any.whl
```

适合：

- 希望以“像正式依赖一样”的方式引用
- 不想使用 editable 模式

## 4.3 按 extras 安装

当前 `pyproject.toml` 已定义：

- `kmcache[redis]`
- `kmcache[fastapi]`
- `kmcache[msgpack]`
- `kmcache[all]`

示例：

```bash
python -m pip install ".[all]"
python -m pip install ".[redis]"
python -m pip install ".[fastapi]"
```

说明：

- 当前项目核心 `dependencies` 为空，框架和 Redis 依赖全部按 extras 安装
- `msgpack` 为可选序列化扩展

## 5. 三种使用模式

## 5.1 仅使用 L1

适合：

- 单实例服务
- 配置缓存
- 小规模热点数据

示例：

```python
from kmcache.backends.local import LocalCacheBackend
from kmcache.config import CacheConfig
from kmcache.manager import CacheManager

cache = CacheManager(
    [LocalCacheBackend()],
    CacheConfig(ttl_jitter=0),
)
```

## 5.2 仅使用 L2

适合：

- 多实例服务
- 需要共享缓存
- 不需要本地 L1 的场景

示例：

```python
from kmcache.backends.redis import RedisCacheBackend
from kmcache.config import CacheConfig, RedisCacheConfig
from kmcache.manager import CacheManager

redis_config = RedisCacheConfig(
    url="redis://127.0.0.1:6379/0",
    key_prefix="kmcache-demo",
)

cache = CacheManager(
    [RedisCacheBackend.from_url(redis_config.url, redis_config)],
    CacheConfig(ttl_jitter=0, redis=redis_config),
)
```

## 5.3 使用 L1 + L2

推荐生产默认模式。

示例：

```python
from kmcache.backends.local import LocalCacheBackend
from kmcache.backends.redis import RedisCacheBackend
from kmcache.config import CacheConfig, RedisCacheConfig
from kmcache.manager import CacheManager

redis_config = RedisCacheConfig(
    url="redis://127.0.0.1:6379/0",
    key_prefix="kmcache-demo",
)

cache = CacheManager(
    [
        LocalCacheBackend(),
        RedisCacheBackend.from_url(redis_config.url, redis_config),
    ],
    CacheConfig(ttl_jitter=0, redis=redis_config),
)
```

## 6. 核心 API

## 6.1 `set`

```python
await cache.set("user:1", {"name": "alice"}, ttl=60, soft_ttl=30)
```

说明：

- `ttl`：硬过期时间
- `soft_ttl`：软过期时间

## 6.2 `get`

```python
user = await cache.get("user:1")
```

未命中时返回 `None`，也可以传默认值：

```python
user = await cache.get("user:1", default={})
```

## 6.3 `get_or_load`

这是最重要的接口。

```python
async def load_user() -> dict[str, str]:
    return {"name": "alice"}

user = await cache.get_or_load(
    "user:1",
    loader=load_user,
    ttl=60,
    soft_ttl=30,
)
```

能力包括：

- miss 回源
- 空值缓存
- stale 返回
- 单进程 single-flight
- 多实例分布式锁
- loader 失败时旧值兜底

## 6.4 `delete`

```python
await cache.delete("user:1")
```

## 6.5 `clear`

```python
await cache.clear()
```

## 6.6 `warmup`

```python
from kmcache.models import WarmupItem

await cache.warmup(
    [
        WarmupItem(
            key="config:site",
            loader=lambda: {"site": "demo"},
            ttl=300,
            soft_ttl=120,
        )
    ]
)
```

## 8. 广播、熔断器、分布式锁

## 8.1 广播通知

启用后，多实例之间通过 Redis Pub/Sub 同步本地 L1 失效。

```python
config = CacheConfig(
    redis=redis_config,
    broadcast=BroadcastConfig(
        enabled=True,
        channel="kmcache:demo:broadcast",
        instance_id="instance-a",
    ),
)
```

## 8.2 熔断器

```python
config = CacheConfig(
    circuit_breaker=CircuitBreakerConfig(
        enabled=True,
        failure_threshold=5,
        recovery_timeout=30.0,
        half_open_max_calls=1,
    )
)
```

说明：

- 后端失败达到阈值后打开熔断
- 已熔断后端后续会被自动跳过
- 指标与日志会记录熔断打开事件

## 8.3 分布式锁

```python
redis_config = RedisCacheConfig(
    url="redis://127.0.0.1:6379/0",
    key_prefix="kmcache-demo",
    lock_timeout=5.0,
    lock_sleep_interval=0.05,
)
```

说明：

- 多实例并发 miss 时，只允许一个实例执行真实 loader
- 其他实例等待 Redis/L2 回填

## 10. FastAPI 集成

## 10.1 生命周期管理

```python
from fastapi import FastAPI

from kmcache.backends.local import LocalCacheBackend
from kmcache.config import CacheConfig
from kmcache.integrations.fastapi import create_cache_lifespan
from kmcache.manager import CacheManager

cache = CacheManager(
    [LocalCacheBackend()],
    CacheConfig(ttl_jitter=0),
)

app = FastAPI(lifespan=create_cache_lifespan(cache))
```

## 10.2 依赖注入

```python
from fastapi import Depends

from kmcache.integrations.fastapi import get_cache
from kmcache.manager import CacheManager


@app.get("/users/{user_id}")
async def get_user(
    user_id: int,
    dependency_cache: CacheManager = Depends(get_cache),
):
    return await dependency_cache.get(f"user:{user_id}")
```

## 10.3 装饰器缓存

```python
from kmcache.integrations.decorators import cached


@cached(
    cache=cache,
    key_builder=lambda user_id: f"user:{user_id}",
    ttl=60,
)
async def load_user(user_id: int) -> dict[str, int]:
    return {"user_id": user_id}
```
